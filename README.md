# kursovarobota
  Система управління базою даних навчального закладу.
  
  Рік виконання: 2025


 Навчальний заклад: Чернівецький Національний Університет імені Юрія Федьковича

  Опис проєкту
  
Цей проєкт є консольною системою керування базою даних навчального закладу.  
Система дозволяє зберігати, редагувати та переглядати інформацію про студентів, викладачів і навчальні предмети.  
Також реалізовано можливість формування груп вивчення предметів, де кожен предмет закріплюється за конкретним викладачем.

   Основні можливості
   
-  Зберігання даних про студентів, викладачів і предмети.  
-  Додавання нових записів (студентів, викладачів, предметів).  
-  Редагування або видалення існуючих записів.  
-  Пошук даних за різними критеріями.  
-  Запис студентів до груп вивчення певного предмета.  
-  Прив’язка предметів до конкретного викладача (один предмет — один викладач).  
-  Збереження та завантаження інформації з файлу.

  Структура системи
  
Програма побудована з використанням принципів **об’єктно-орієнтованого програмування (ООП)**.

   Основні класи :
- `Student` — зберігає інформацію про студента (ПІБ, група, ID).  
- `Teacher` — містить дані про викладача (ПІБ, кафедра, предмет).  
- `Subject` — описує навчальний предмет (назва, закріплений викладач).  
- `IPrint` — інтерфейс, який визначає метод для форматованого виведення інформації про об’єкти.  
- `DatabaseManager` — головний клас, який управляє колекціями студентів, викладачів і предметів, реалізує збереження/читання з файлу та логіку взаємодії.

 Технології
- Мова програмування: C++  
- Парадигма: Об’єктно-орієнтоване програмування  
- Основні концепції: наслідування, інтерфейси, поліморфізм, інкапсуляція  
- Робота з файлами: текстові файли (`.txt`)  
- Розумні вказівники: `std::shared_ptr`, `std::unique_ptr`  
- Обробка винятків: через `try`, `catch`  
- Організація коду: розділення на `.h` та `.cpp` файли
